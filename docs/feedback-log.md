구현 경로 (설계 확정, 착수는 유통 큐 소진 후):

공고 이미지 — B안: 서버 측 비전 전사 (채택)
- 구조: 이미지 → 비전 모델 전사만("적힌 글자를 그대로 받아 적어라") → 추출 텍스트를 기존 입력창에 삽입 → 유저가 확인·수정 후 기존 제출. 이미지→요구사항 직행 금지 — 측정 안 된 새 파이프라인이 생긴다. 전사만 시켜야 동결 파이프라인이 유지된다
- 진입로 2개: ① 붙여넣기 통합(paste 이벤트에서 클립보드 이미지 감지 → OCR → 커서 위치에 텍스트 삽입. 데스크톱 주력) ② 업로드/드래그 버튼(모바일 폴백 — 모바일은 이미지 클립보드 붙여넣기가 불안정). 두 경로 모두 같은 OCR 엔드포인트, 같은 입력창으로 수렴
- 모델: 기존 스택(OpenAI)의 비전 지원 모델로 통일 — 키·과금·에러 처리 단일 경로. 무료 티어류 학습 이용 이슈 원천 배제
- 안전선: 이미지 디스크 미저장(메모리 전사 후 폐기 — "저장 안 함" 약속의 연장), 파일 크기 상한, OCR 호출 레이트리밋 포함, 비용 실측 후 전역 캡 재계산
- 착수 전 스파이크 (필수 게이트): 실제 이미지 공고 5건을 비전 전사 → 원본과 눈 대조. 전사 품질이 부족하면 착수 자체를 접는다. 판단 비용 ~30분, 몇십 원
- 성공 지표 (사전 정의): 방문→실행 전환율, 기준선 9% (07-18 digest)
